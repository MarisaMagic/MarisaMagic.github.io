A 東方 Project 人気投票

出题人：MarisaMagic

题意简述

有 nnn 个角色，次序编号从 1∼n1 \sim n1∼n。共 mmm 个人参与投票，每个人投票情况用长度为 nnn 的 ox 字符串表示，对于字符串第 k 个字符，若为 o 表示给第 k 个角色投了 111 票，x 表示未投。按照角色获得票数从大到小排序（若票数相同，按角色次序编号从小到大排序）。

解题思路 （结构体排序）

定义一个结构体，存储每个角色的名称 namenamename、获得的票数 scorescorescore 以及次序编号 ididid。遍历每个人的投票情况，更新每个角色的票数，最后根据题意排序输出结果即可。

B 恋色宝石

出题人：MarisaMagic

题意简述

给定一个长度为 nnn 的序列 aaa，求 max⁡1≤i≤j≤n{aj−ai}\max_{1 \le i \le j \le n} \{ a_j - a_i \}max1≤i≤j≤n​{aj​−ai​}。

解题思路 （动态规划，前缀最小值）

暴力的思路即从 1≤j≤n1 \le j \le n1≤j≤n 枚举 aja_jaj​，对于当前的 aja_jaj​ 再枚举其前面所有位置的 aia_iai​（1≤i≤j1 \le i \le j1≤i≤j）。对于每个 aja_jaj​ 枚举得到 max⁡1≤i≤j{aj−ai}\max_{1 \le i \le j} \{ a_j - a_i \}max1≤i≤j​{aj​−ai​}，最后求得这 nnn 个 max⁡1≤i≤j{aj−ai}\max_{1 \le i \le j} \{ a_j - a_i \}max1≤i≤j​{aj​−ai​} 最大值即为答案。

可以发现，对于每个 aja_jaj​ 来说，max⁡1≤i≤j{aj−ai}\max_{1 \le i \le j} \{ a_j - a_i \}max1≤i≤j​{aj​−ai​} 的最优解 只会在前面最小的 aia_iai​ 上取到，即 min⁡1≤i≤jai\min_{1 \le i \le j}a_imin1≤i≤j​ai​。

故我们不用在枚举 aja_jaj​ 之后再枚举 aia_iai​。只需要在枚举 aja_jaj​ 时，边更新 前缀最小值，边更新答案。时间复杂度为 O(n)\mathcal{O}(n)O(n)。

C 上海？蓬莱？歌利亚！

出题人：MarisaMagic

题意简述

给定两个字符串 ppp 和 qqq，取出 ppp 的一个非空前缀和 qqq 的一个非空后缀并前后连接得到一个字符串。按这种方式共可以得到 多少种 不同的字符串。

解题思路 （字符串，模拟，乘法原理）

假设 ppp 的长度为 nnn，qqq 的长度为 mmm。取 ppp 的非空前缀共有 nnn 种方式，取 qqq 的非空后缀共有 mmm 种方式，显然，不考虑重复情况下可以构成 n×mn \times mn×m 个字符串。接下来考虑去重。

假设有两个字符串分别为 “abc” 和 “dbc”，取 前缀 “ab” 和 后缀 “c” 与 前缀 “a” 和 后缀 “bc” 的所构成的字符串一样，均为 “abc”。 假设有一个字符 chchch 同时在 ppp 和 qqq 中出现，ppp 中 chchch 之前的非空前缀为 pprep_{pre}ppre​，qqq 中 chchch 的非空后缀为 qsufq_{suf}qsuf​。可以发现取 前缀 ppre+chp_{pre} + chppre​+ch 和 后缀 qsufq_{suf}qsuf​ 、前缀 pprep_{pre}ppre​ 和 后缀 ch+qsufch + q_{suf}ch+qsuf​ 两种情况重复了。

因此，我们需要统计每个字符分别在 ppp 和 qqq 中出现的次数，根据乘法原理，假设一个字符 chchch 在 ppp 中出现了 cntpcnt_pcntp​ 次，在 qqq 中出现了 cntqcnt_qcntq​ 次，那么这个字符 chchch 会造成的重复次数为 cntp×cntqcnt_p \times cnt_qcntp​×cntq​。

故只需要统计 262626 个字符分别在 ppp 和 qqq 中出现的次数，最后减去每个字符造成的重复次数即可。

需要注意的是，我们所取前缀和后缀必须是非空的。因此在统计 ppp 中每个字符出现次数时，需要忽略第一个位置的字符；在统计 qqq 中每个字符出现次数时，需要忽略最后一个位置的字符。

时间复杂度为 O(n+m)\mathcal{O}(n + m)O(n+m)。注意数据范围，需要开 long long。

D 弥林的防御布局

出题人：South_Orange

题意简述

判断一个点与已知三角形的位置关系。

解题思路 （计算几何）

当给定点与三角形某个端点相同时，点在端点上

给定点 ppp 在三角形的边上时，要求 ppp 和线段 vvv 的一个端点连边，和 vvv 本身的夹角为 0，即 ppp 在直线 vvv 上，且 ppp 和两端点形成平角，也就是说 ppp 在两端点之间，点与线段的夹角可通过叉积判断。

给定点在三角形内部时，该点应在三角形的三个点形成的三个向量的同一侧，点与向量的位置关系可通过叉积判断。

附录见 Geo.cpp，可作为基础二维计算几何模板进行参考学习。

其他做法可见 P1355 神秘大三角 题解，主要采用海伦公式。

E 姆Q的魔导书

出题人：MarisaMagic

题意简述

给定一个可变序列 a1,a2,…,ana_1, a_2, \ldots, a_na1​,a2​,…,an​，有 qqq 次操作，分为两种：

  • 1 l r x：询问区间 [l,r][l, r][l,r] 内数字 xxx 的数量；
  • 2 x：将 111 个 xxx 加入到序列的末尾。

解题思路 （离散化，二分）

主席树（可持久化线段树） 可以解决区间数字个数询问的问题，但是本题无需使用主席树。

初始序列长度和操作次数加起来最多 3×1053 \times 10^53×105，故序列中数字种类数不超过 3×1053 \times 10^53×105。考虑离散化存储每个数字出现的所有下标，可以通过 unordered_map<int, vector<int>> 来实现，其中 unordered_map 的键为数字，值为可变长数组，存储的是每个数字出现的下标。

  • 对于操作 1 l r x：

    由于存储的下标是递增的，故我们可以通过二分来查找范围。对于当前询问的数字 xxx，我们可以借助 lower_bound 找到 xxx 所有下标中 第一个大于等于 lll 的下标位置 plp_lpl​，借助 upper_bound 找到 xxx 所有下标中 最后一个大于 rrr 的下标位置 prp_rpr​。最终 [l,r][l, r][l,r] 区间内 xxx 的个数即 pr−plp_r - p_lpr​−pl​。

  • 对于操作 2 x：

    记录插入末尾下标 idxidxidx，每次插入将 idxidxidx 插入到对应数字下标数组中。插入完成后 idx+=1idx += 1idx+=1。

F 橙

出题人：MarisaMagic

题意简述

给定一颗 nnn 个节点的无向树，通过一条边需要 111 单位时间，有 mmm 个节点设有传送门，mmm 个节点之间可瞬间互相传送。有 qqq 次询问，每次询问两点通过所需最短时间。

解题思路 （最近公共祖先 LCA，多源bfs）

如果 mmm 为 000，可以先用 倍增算法求最近公共祖先 LCA。预处理求得根节点到其他点距离 ddd，树上两点 x,yx, yx,y 最短距离即 dx+dy−2×dancd_x + d_y - 2 \times d_{anc}dx​+dy​−2×danc​，其中 ancancanc 为 x,yx, yx,y 的最近公共祖先。

如果 mmm 不为 000，那么可以选择从 xxx 花费一段时间走到距离其最近的传送门 transxtrans_xtransx​，瞬间传送到距离 yyy 最近的传送门 transytrans_ytransy​，最后再花费一段时间从 transytrans_ytransy​ 走到 yyy，即 x→transx→transy→yx \rightarrow trans_x \rightarrow trans_y \rightarrow yx→transx​→transy​→y。此方案的答案即 xxx 到最近传送门 transxtrans_xtransx​ 的距离 + yyy 到最近传送门 transytrans_ytransy​ 的距离

边都是双向的，只需要 多源01bfs 预处理每个传送门到达其他节点的最短距离 distdistdist。最终每次询问 x,yx, yx,y 的答案即 min⁡(dx+dy−2×danc，distx+disty)\min{ (d_x + d_y - 2 \times d_{anc}， dist_x + dist_y) }min(dx​+dy​−2×danc​，distx​+disty​)。

预处理 LCA 时间复杂度为 O(nlog⁡n)\mathcal{O}(n\log{n})O(nlogn)，每次询问复杂度为 O(log⁡n)\mathcal{O}(\log{n})O(logn)。总时间复杂度为 O((n+q)log⁡n)\mathcal{O}((n + q)\log{n})O((n+q)logn)。

G 东方 × 地铁

出题人：MarisaMagic

题意简述

给定一个 n×mn \times mn×m 的网格图，行从 1∼n1 \sim n1∼n 编号，列从 1∼m1 \sim m1∼m 编号，每个点可用所在行编号 xxx 与所在列编号 yyy 表示为 (x,y)(x, y)(x,y)。

点 (i,j)(i, j)(i,j) 与 (i,j+1)(i, j + 1)(i,j+1) 间连有一条权值为 aia_iai​ 的边，其中 1≤i≤n,1≤j<m1 \le i \le n, 1 \le j < m1≤i≤n,1≤j<m。

点 (i,j)(i, j)(i,j) 与 (i+1,j)(i + 1, j)(i+1,j) 间连有一条权值为 bjb_jbj​ 的边，其中 1≤i<n,1≤j≤m1 \le i < n, 1 \le j \le m1≤i<n,1≤j≤m。

求这个网格图的最小生成树。

解题思路 （贪心，kruskal 算法思想）

如果强行建边用 kruskal 求最小生成树，边的数量达到 O(n×m)O(n \times m)O(n×m) 级别，显然不可取。

但是，一行或一列中边的长度一样，不同长度的边最多 n+mn + mn+m 种。我们可以仿照 kruskal\text{kruskal}kruskal 求最小生成树的贪心思想，先分别对所有行边权 aaa 和 列边权 bbb 进行从小到大排序，然后 贪心地选取边，并 维护生成树中不出现环。

我们可以先选取 所有行边权中最小的边权 a1a_1a1​，然后全部加入到最小生成树；然后选取 所有列边权中最小的边权 b1b_1b1​，也可以全部加入到最小生成树，并不会出现环。这一步加入到最小生成树的总长度为 (m−1)×a1+(n−1)×b1(m - 1) \times a_1 + (n - 1) \times b_1(m−1)×a1​+(n−1)×b1​。

对于接下来选取的行边权和列边权，我们每次对比当前的最小行边权与最小列边权的大小，谁更小先选谁。同时还要统计此前选取的行边权的个数 cntrcnt_rcntr​ 和列边权的个数 cntccnt_ccntc​。

  • 若当前最小行边权 <<< 最小列边权：

    可以发现这一行中已经有 cntccnt_ccntc​ 个点已经加入到最小生成树中。由于当前边权最小，我们需要尽可能多地选取这一行的边且保证不出现环。我们可以选择在 cntccnt_ccntc​ 个点中的每两个相邻点之间断开一条边，这样可以保证不会出现环。故我们当前这一行连的边条数为 (m−cntc)(m - cnt_c)(m−cntc​)。

    由于选择了一个行边权，因此还需要更新 cntr+=1cnt_r += 1cntr​+=1。

  • 若当前最小行边权 ≥\ge≥ 最小列边权：

    与上面同理，这一列有 cntrcnt_rcntr​ 个点已经加入到最小生成树中。为了最小生成树总长度尽量小且保证不出现环，我们当前这一列连的边条数为 (n−cntr)(n - cnt_r)(n−cntr​) 即可。

    同理，还需要更新 cntc+=1cnt_c += 1cntc​+=1。

最后当总边数 cnt=n×m−1cnt = n \times m - 1cnt=n×m−1 时退出循环。总时间复杂度为 O((n+m)log⁡(n+m))\mathcal{O}((n + m)\log{(n + m)})O((n+m)log(n+m))。
