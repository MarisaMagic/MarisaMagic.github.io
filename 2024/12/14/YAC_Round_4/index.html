B. 灵梦的香火钱

题目描述

给定 nnn 个元素，将这 nnn 个元素两两进行配对，两个元素配对之后得到的值为对应数字首尾拼接得到的数字。求出最多能够凑出多少 333 的倍数。https://www.luogu.com.cn/problem/T425162

解题思路 数学

我们可以假设当前配对的两个数字分别为 x1x_1x1​ 和 x2x_2x2​。我们可以进行对 333 取模运算后，可以将两个数字分解后表示如下：

x1=k1×3+r1x_1 = k_1 \times 3 + r_1 x1​=k1​×3+r1​

x2=k2×3+r2x_2 = k_2 \times 3 + r_2 x2​=k2​×3+r2​

其中，rrr 为 xxx 对 333 取模后得到的余数。

我们将 x1x_1x1​ 的尾部和 x2x_2x2​ 的首部进行拼接，同时我们可以假设 x2x_2x2​ 的数字长度为 mmm，那么也就是说拼接后得到的数字将会是 x1×10m+x2x_1 \times 10^m + x_2x1​×10m+x2​，即：

(k1×3)×10m+r1×10m+k2×3+r2(k_1 \times 3) \times 10^m + r_1 \times 10^m + k_2 \times 3 + r_2 (k1​×3)×10m+r1​×10m+k2​×3+r2​

取模运算是符合加法和乘法运算的，故我们对当前拼接后得到的数字的各个部分进行对 333 取模:

(k1×3)×10m mod 3=k1×3 mod 3=0(k_1 \times 3) \times 10^m \bmod 3 = k_1 \times 3 \bmod 3 = 0 (k1​×3)×10mmod3=k1​×3mod3=0

r1×10m mod 3=r1r_1 \times 10^m \bmod 3 = r_1 r1​×10mmod3=r1​

k2×3 mod 3=0k_2 \times 3 \bmod 3 = 0 k2​×3mod3=0

r2 mod 3=r2r_2 \bmod 3 = r_2 r2​mod3=r2​

因此，拼接后得到数字对 333 取模后即：r1+r2r_1 + r_2r1​+r2​。（会发现，拼接得到数字模3的结果和相加是一样的）

那么，要使得拼接后为 333 的倍数，显然 r1+r2=0r_1 + r_2 = 0r1​+r2​=0 或者 r1+r2=3r_1 + r_2 = 3r1​+r2​=3，所以要么 r1r_1r1​ 和 r2r_2r2​ 都为 000；要么 r1r_1r1​ 和 r2r_2r2​ 中一个为 111 另一个为 222。

也就是余数为 000 的只可以和余数同样是 000 的相匹配；余数为 111 或 222 的，只可以和 222 或 111 匹配。对于前者，由于我们不匹配的数字不算，所以 r1+r2=0r_1 + r_2 = 0r1​+r2​=0 最多可以匹配出 ⌊cnt02⌋\left \lfloor \frac{cnt_0}{2} \right \rfloor⌊2cnt0​​⌋ 个 333 的倍数；对于后者，则为 min⁡(cnt1,cnt2)\min(cnt_1, cnt_2)min(cnt1​,cnt2​) 。

故我们只需要统计每个余数有多少个元素，最后输出答案为 ⌊cnt02⌋+min⁡(cnt1,cnt2)\left \lfloor \frac{cnt_0}{2} \right \rfloor + \min(cnt_1, cnt_2)⌊2cnt0​​⌋+min(cnt1​,cnt2​) 。

C. 青春异或少年不会遇到按位与学姐

题目描述

给定一个正整数 xxx，找出最小的满足以下条件的正整数 yyy：

  • xxx and\text{and}and y>0y > 0y>0

  • xxx xor\text{xor}xor y>0y > 0y>0

「YAC Round 4」青春异或少年不会遇到按位与学姐

解题思路 位运算 + 贪心

对于第一个条件，两个数字 xxx 和 yyy 按位与大于 000，显然 xxx 和 yyy 的对应二进制位中至少需要有 111 位同时是 111，那么为了让 yyy 尽可能小，我们取 xxx 最后一位 111 即前面位的所有 000 即可。例如 10=(1010)210 = (1010)_210=(1010)2​，此时取 2=(10)22 = (10)_22=(10)2​ 即可，也就是 lowbit(10)\text{lowbit} (10)lowbit(10)。 浅谈lowbit运算 - Seaway-Fu - 博客园

对于第二个条件，两个数字 xxx 和 yyy 按位异或大于 000，那么我们至少需要有一位二进制位不同。在第一个条件下我们取了 y′=lowbit(x)y^{'} = \text{lowbit}(x)y′=lowbit(x) 。如果此时 y′≠xy^{'} \not = xy′=x，那么就取此时的 y=y′y = y^{'}y=y′ 即可。

如果 y′=xy^{'} = xy′=x，说明此时取的 y′y^{'}y′ 和 xxx 每一位都一样，为了使得 yyy 尽量小，我们只需要让此时 y′y^{'}y′ 的最低位的 000 为 111 即可。一般情况下，我们只要使得 y′y^{'}y′ 的最低位变为 111 即可，也就是 取 y=y′+1y = y^{'} + 1y=y′+1 即可。

但是，当 x=1x = 1x=1 时，需要特殊判断一下，此时为了满足第一个条件，y′=1y^{'} = 1y′=1，为了满足第二个条件，由于最低位取 111，故只能使得第二位为 111，也就是取 y=3y = 3y=3。

D. K-ON!

题目描述

给定 nnn 个社团，每个社团有一个表演时间 aia_iai​，按照下标顺序上台表演，舞台可以同时容纳多个是社团进行表演（即舞台有一个大小）。给定一个总时间限制 ttt，在所有社团表演总时间不超过 ttt 的情况下，使得舞台大小最小化。「YAC Round 4」K-ON!

解题思路 二分 + 堆

可以发现舞台越大，总时间必定越少；反之亦然。

显然问题具有二段性，因此我们考虑二分答案（即舞台大小）。

假设二分时的舞台大小为 kkk。在二分的判断函数中，我们需要每次取出当前 kkk 个社团中最早表演完的一个社团让其下场，同时按照编号顺序让下一个要上场的社团上场开始表演。故我们可以用堆（优先队列）维护当前 kkk 个社团表演结束时间，因为要取最小值，所以我们选用的是小根堆来进行模拟。

如果在模拟过程中，当前要上场的社团表演时间大于总时间限制时，就返回 false\text{false}false；模拟到最后都没有出现超过时间限制的社团，那么就返回 true\text{true}true 。

PS：题目保证一定有解，所以不用担心一开始就出现无法表演结束的社团。

E. 早苗的机动战士高达梦

题目描述

给定一个图，图中构成多个连通块。先输出当前图中的连通块个数，然后给定 kkk 次询问，每次去除一个点及其邻接的边，问去除当前这个点时图中有多少连通块。「YAC Round 4」早苗的机动战士高达梦

解题思路 并查集

我们可以用并查集维护连通块个数。

如果我们正向考虑这个问题，那么我们每次询问都需要重新建立关系，然后得到去除当前点后的连通块个数，这显然是会超时的。

故我们考虑逆向思考这个问题。预先将所有的边都存起来。我们可以先得到经过最后一次操作后，图中还有多少连通块。然后，每一次还原一个点，相当于每次加入一个点及其邻接边，逆向还原每次操作之前图中的连通块个数。

答案存储在数组中，最后输出即可。

F. 爱丽丝送给魔理沙的手链

题目描述

假设 bbb 为蓝色，ggg 为金色。给定一个长度 nnn，求出用 bbb 和 ggg 可以构造出的相邻元素不能同时为金色，即不能同时为 ggg（或者说相邻元素至少有一个为蓝色，即至少有一个为 bbb） 的长度为 nnn 的 环状 字符串方案数。「YAC Round 4」爱丽丝送给魔理沙的手链

解题思路 dp + 矩阵加速

我们可以先考虑一下 n≤106n \le 10^6n≤106 时如何解决这个问题。显然，我们可以用动态规划解决这个子问题。

用 dp[i][0]dp[i][0]dp[i][0] 表示构造的长度为 iii 且第 iii 个宝石选用 金色宝石 的方案数； dp[i][1]dp[i][1]dp[i][1] 表示构造的长度为 iii 且第 iii 个宝石选用 蓝色宝石 的方案数。

题中要求相邻的两个不能同时为金色。对于 dp[i][0]dp[i][0]dp[i][0] ，第 iii 个选择了金色，那么前一位只能用蓝色；对于 dp[i][1]dp[i][1]dp[i][1]，第 iii 个选择了蓝色，那么前一位选择蓝色或金色都可以。 故状态转移方程如下：

{dp[i][0]=dp[i−1][1]dp[i][1]=dp[i−1][0]+dp[i−1][1]\begin{cases} dp[i][0] = dp[i - 1][1] \\\\ dp[i][1] = dp[i - 1][0] + dp[i - 1][1] \end{cases} ⎩⎨⎧​dp[i][0]=dp[i−1][1]dp[i][1]=dp[i−1][0]+dp[i−1][1]​

由于是 环状 的，我们还需要进行一下初始化的分类讨论。

  • 如果第 111 个选择金色宝石，那么第 nnn 个宝石只能选择蓝色宝石（因为环形连接处也需要满足条件），故此情况答案为 dp[n][1]dp[n][1]dp[n][1]

  • 如果第 111 个选择蓝色宝石，那么第 nnn 个宝石两种颜色都可以选择，故此情况答案为 dp[n][0]+dp[n][1]dp[n][0] + dp[n][1]dp[n][0]+dp[n][1]
